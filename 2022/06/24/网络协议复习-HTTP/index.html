HTTP

  1. 基于TCP
  2. 2.0版本对包头进行压缩
    1. 1.1多请求串行
    2. 2.0请求并行
  3. keep-alive 保持连接不会立刻中断tcp连接，请求复用

HTTPS

  1. https的通信流程是为了通过非对称加密来生成对称加密使用的参数
  2. 流程
    1. 客户端生成随机数
    2. 服务端返回随机数
    3. 审核服务端证书
    4. 审核客户端证书
    5. 生成pre-master
    6. 服务端+客户端+pre-master 算出对称密钥
    7. 测试参数- c->s
    8. 测试参数- s->c
    9. 完成
  3. 包内包含时间戳和随机数，确保不会被重放和篡改
  4. SSL在应用层和传输层之间

QUIC

  1. HTTP3.0
  2. 基于udp
  3. 自定义连接机制
  4. 自定义重传机制
  5. 无阻塞的多路复用
  6. 自定义流量控制
