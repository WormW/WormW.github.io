GoLang基础-内存模型
Post：2022-06-29 16:26:15 Tags：/ golang / 基础 / 内存模型 / memory model / Visit：
  1. 为了保证goroutine读取串行化，需要使用channel或者同步原语，比如sync或者sync/atomic，mutex
  2. happen-before
    1. 同一个goroutine中的读写顺序是按照程序中的顺序执行的
    2. 不同goroutine中的可能看到不同的执行顺序
    3. 内存重排 memory recordering
    4. 抚平内核内存硬盘之间的宿绿茶 l1 l2 l3 cache store buffer
    5. 由于store buffer（类似数据还在缓存，没能读到，直接从memory里读取）
    6. 通过barrier或者fence 指令 保证类似锁的机制
    7. 单个goroutine中没有并发
    8. 变量v的零值初始化在内存模型中表现的与写操作相同
    9. 对大于single machine word的变量的读写操作表现的像以不确定顺序对多个single machine word的变量的操作
    10. sync.Once是一种全局的类似singleflight的机制，但是singleflight的粒度更低，通过key去控制调用的请求
    11. 需要用锁的机制保证先行发生，即按照正确的语义去控制变量的写
    12. 在goroutine的嵌套中对于非原子化的赋值要慎重（主要针对goroutine共享变量或者潜逃goroutine），可能会导致取值异常
