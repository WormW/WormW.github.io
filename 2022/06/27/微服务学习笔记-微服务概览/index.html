微服务学习笔记-微服务概览
Post：2022-06-27 23:05:20 Tags：/ 微服务 / Visit：
  1. 微服务发展历程
  2. 单体架构
    1. 使用模块化逻辑划分
    2. 最终打包成大型单体
    3. 化繁为简分而治之
  3. 微服务起源
    1. 特点
      1. SOA面向服务的架构模式
      2. 小即是美
      3. 单一指责
      4. 尽可能早地创建原型
      5. 可移植性比效率更重要
    2. 运维和开发本质上对于上线是有冲突的，运维需要保持尽量的稳定，开发需要尽快的上线，对于单体架构存在更大的冲突，上线日会造成比较大的麻烦
    3. 要具备通信的通用性
    4. 定义
      1. 原子服务（拆分了DB增加了系统的整体吞吐性能）
      2. 独立进程
      3. 隔离部署（一个物理机也可以部署多个隔离pod）（对部署多个微服务 降低crash影响）
      4. 去中心化服务治理
      5. 基础设施建设、复杂度高（缺点）
    5. 不足
      1. 增加了固有的复杂性，需要处理服务不可用和服务传递速度慢的问题
      2. 分布式事务
      3. 测试单一应用变复杂
      4. 服务见的依赖，导致单一应用影响多个服务
      5. 运维基础设施挑战高
  4. 组件服务化（多个微服务组成一个usercase）
    1. kit:需要一套微服务基础库（框架内包）
    2. service: 由 业务代码+kit+第三方依赖组成
    3. RPC + message queue：轻量级通讯
  5. 去中心化
    1. 数据去中心化（缓存数据隔离）
    2. 治理去中心化（避免中心热点）
    3. 技术去中心化（不绑定语言）
  6. 基础设施自动化
    1. cicd：gitlab +gitlab hooks +kubernetes
    2. Testing：测试环境、单元测试、API自动化测试
    3. 在线运行时：kubernetes，以及一系列prometheus、ELK、control panel
  7. 可用性&兼容性设计
    1. Design For Failure
    2. 隔离
    3. 超时控制
    4. 负载保护
    5. 限流
    6. 降级
    7. 重试
    8. 负载均衡
  8. 发送时保守，接受时开放【伯斯塔尔法则】尽量少传数据，接收时可以接受冗余
