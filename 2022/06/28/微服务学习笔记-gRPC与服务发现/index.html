微服务学习笔记-gRPC与服务发现
发布于：2022-06-28 10:36:37 标签：/ 微服务 / 访问：

  1. gRPC

    1. 特点

      1. 多语言：语言中立
      2. 轻量级、高性能：序列化支持Protocol Buffer和JSON
      3. 可插拔
      4. IDL：基于文件定义服务，通过proto3工具生成指定语言的数据结构、服务端接口以及客户端Stub
      5. 设计理念
      6. 移动端：基于http/2设计，支持双向流、消息头压缩、单TCP的多路复用、服务端推送等特性
      7. 服务而非对象、消息而非饮用：微服务间粗粒度消息交互
      8. 负载无关
      9. 流：streaming api
      10. 阻塞式和非阻塞式
      11. 元数据交流
      12. 标准化状态码

    2. 先做标准化，再考虑性能

    3. HealthCheck 标准的健康监测协议

      1. 用于外挂的容器健康监测或者流量监测（readiness liveness）
      2. 平滑发布
        1. k8s 向discovery发起注销
        2. k8s 向app发送sigterm
        3. 最长两个心跳骤起内退出
        4. k8s退出超时 sigkil

  2. 服务发现

    1. 客户端发现

      1. 直连
      2. 少一次网络跳转
      3. 需要客户端实现知道服务发现和转发逻辑

    2. 服务端发现（服务实例在服务注册表上被注册和注销）

      1. 多一次网络跳转
      2. 性能有所损失

    3. 微服务核心是去中心化，使用客户端发现模式

    4. zookeeper、eureka

    5. 自保护机制

    6. 心跳机制

    7. 少量节点负责写，大量节点负责读的模式（Eureka2.0）
