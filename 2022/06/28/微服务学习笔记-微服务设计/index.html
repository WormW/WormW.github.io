微服务学习笔记-微服务设计
Post：2022-06-28 00:25:03 Tags：/ 微服务 / Visit：
  1. API Gateway
    1. 直接暴露微服务问题
      1. 如果没有统一网关，会导致无法下掉老旧接口
      2. 客户端到微服务强耦合
      3. 多次请求，客户端聚合数据，工作量大，延迟高（影响客户端交付速度）
      4. 把对接复杂度聚焦于端
      5. 适配复杂度下移到微服务本身了
    2. 解决方案app-interface（BFF）
      1. 胶水层负责组装数据
      2. 轻量交互，协议精简、聚合
      3. 差异服务，数据裁剪以及聚合、针对终端定制API
      4. 动态升级
      5. 沟通效率提高
    3. BFF属于 single point of failure，严重代码缺陷或者流量洪峰可能引发集群宕机
      1. 拆分BFF，到具体模块
      2. 升级框架中间件变困难
      3. 跨横切面cross-cutting concerns 直接继续继续分层
        1. 中间件功能使用api-gateway
    4. 流量流向 端->api gateway -> bff -> microservice
  2. 微服务划分
    1. 业务职能
    2. DDD的限界上下文bounded context
      1. 业务边界抽象
      2. 抽象业务域
  3. CQRS
    1. 命令端和查询端解耦
    2. canal->kafka->清洗数据后将数据写入查询用库
  4. api-gateway负责统一鉴权
  5. 服务内部区分身份认证和授权（RBAC）
    1. full trust
    2. half trust
    3. zero trust
  6. RPC鉴权
    1. 证书
    2. token
